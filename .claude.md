# Accessible Astro Starter

## Project Overview

This is a modern, accessibility-focused static site generator (SSG) starter template built with Astro. It serves as the flagship starter theme of the Accessible Astro ecosystem, designed for building inclusive blogs, portfolios, and content-driven websites that comply with WCAG 2.2 AA standards and are ready for the European Accessibility Act (EAA).

**Version:** 4.3.1
**License:** MIT
**Site URL:** https://accessible-astro-starter.incluud.dev
**Repository:** https://github.com/incluud/accessible-astro-starter

## Technology Stack

### Core Framework
- **Astro 6.0.8** - Static site generator with excellent performance
- **TypeScript** - Strict mode enabled with configured path aliases
- **Node.js v22.12.0+** - Required runtime version (Astro 6 requires Node 22+)

### Styling & Design
- **Tailwind CSS 4.1.13** - Utility-first CSS framework via Vite plugin
- **SCSS 1.92.1** - Preprocessor with OKLCH color system
- **Logical CSS properties** - Modern CSS with custom properties
- **Atkinson Hyperlegible** - Primary font for accessibility

### Components & Content
- **accessible-astro-components** (v5.0.1) - Comprehensive component library (25+ accessible components)
- **@astrojs/mdx** (v5.0.2) - MDX support for enhanced markdown
- **@astrojs/sitemap** (v3.7.1) - Automatic XML sitemap generation

### Icons & Optimization
- **astro-icon** (v1.1.5) - Icon system with Lucide icon set (requires `@iconify` overrides, see Migration Log)
- **astro-compress** (v2.4.0) - HTML/CSS/JS compression for production

### Code Quality
- **ESLint 9.35.0** with strict accessibility rules
  - `eslint-plugin-jsx-a11y` - Accessibility linting
  - `eslint-plugin-astro` - Astro-specific linting
  - `@typescript-eslint` - TypeScript linting
- **Prettier 3.6.2** with plugins for Astro, Tailwind, and CSS ordering

## Best Practices

1. **Always use semantic HTML** - Proper heading hierarchy, landmarks, and ARIA labels
2. **Test keyboard navigation** - Ensure all interactive elements are keyboard accessible
3. **Verify color contrast** - Use the built-in contrast checker tool
4. **Use path aliases** - Import from `@components`, `@layouts`, `@assets` for cleaner code
5. **Follow WCAG 2.2 AA** - Reference the accessibility statement for guidelines
6. **Leverage accessible components** - Use the accessible-astro-components library
7. **Test with screen readers** - Verify ARIA labels and alt text
8. **Optimize images** - Use Astro's image optimization features
9. **Keep content in collections** - Use MDX for rich, maintainable content
10. **Run linting regularly** - ESLint will catch accessibility issues early

## Notes for AI Assistants

When working with this codebase:
- **Prioritize accessibility** in all implementations
- **Use the accessible-astro-components library** when available
- **Follow TypeScript strict mode** - no type assertions without good reason
- **Respect path aliases** - use `@components/*`, `@layouts/*`, etc.
- **Maintain WCAG 2.2 AA compliance** in all changes
- **Test keyboard navigation** for new interactive elements
- **Use semantic HTML** and proper ARIA labels
- **Follow the existing code style** - Prettier configuration is strict
- **Consider color contrast** when modifying styles
- **Document accessibility features** in new components
- **Use `astro/zod` for schema imports** - `z` must be imported from `astro/zod`, not from `astro:content`
- **Be aware of `astro-icon` overrides** - The project uses npm overrides for `@iconify/tools` and `@iconify/utils` to maintain compatibility (see Migration Log)

## Migration Log: Astro 5 → Astro 6 (2026-03-24)

### Versions Upgraded

| Package | Before | After |
|---------|--------|-------|
| astro | 5.13.7 | 6.0.8 |
| @astrojs/mdx | 4.3.5 | 5.0.2 |
| @astrojs/sitemap | 3.5.1 | 3.7.1 |
| @astrojs/partytown | 2.1.4 | Removed (unused) |
| astro-compress | 2.3.8 | 2.4.0 |
| .nvmrc | v20.5.1 | v22.12.0 |

### Changes Made

1. **`src/content.config.ts`** — Changed `import { defineCollection, z } from 'astro:content'` to separate imports: `defineCollection` from `astro:content` and `z` from `astro/zod`. Required because Astro 6 uses Zod 4 and removed the `z` re-export from `astro:content`.

2. **`astro.config.mjs`** — Changed `server.ws.send()` to `server.hot.send()` in the custom Vite plugin for workspace symlink reloading. Both APIs work in Vite 7, but `server.hot` is the recommended per-environment API going forward.

3. **`package.json`** — Added `overrides` section to pin `@iconify/tools@5.0.5` and `@iconify/utils@3.1.0`. This is needed because `astro-icon@1.1.5` bundles older versions of these packages that are incompatible with Astro 6/Vite 7. The maintainer hasn't released an update (last commit Dec 2024), but PR #278 exists with the fix.

4. **`package.json`** — Updated `astro-compress` to v2.4.0 which fixes an incompatibility where v2.3.x bundled `astro@5` as a direct dependency, causing conflicts with `@astrojs/internal-helpers` version mismatch.

5. **`.nvmrc`** — Updated from `v20.5.1` to `v22.12.0`. Astro 6 requires Node 22+ (dropped Node 18 and 20 support).

6. **Removed `@astrojs/partytown`** — Was listed in `devDependencies` but never configured in `astro.config.mjs` integrations and not imported anywhere in the source code.

### Predicted Problems vs Actual Results

| Predicted Problem | Severity | Actually Occurred? |
|---|---|---|
| `z` import from `astro:content` breaking | Critical | Prevented — fixed before upgrade |
| `server.ws` removed in Vite 7 | Critical | **No** — Vite 7 kept backward compat; migrated anyway |
| `server.moduleGraph` API changed | Critical | **No** — Vite 7 kept backward compat |
| `astro-icon` incompatible | Critical | **Yes** — fixed via `@iconify` overrides |
| `astro-compress` incompatible | High | **Yes** — fixed by updating to v2.4.0 |
| `.nvmrc` blocking CI/CD | Moderate | Prevented — fixed before upgrade |
| `@astrojs/mdx` needs v5 | Moderate | Resolved automatically by `@astrojs/upgrade` |
| Sass legacy API issues | Low | **No** — project already uses modern `@use` syntax |
| Heading ID changes in MDX | Low | **No** — no anchor links in project MDX files |
| Script/style order changes | Low | **No** — no visible regressions |

### Known Warnings (Non-blocking)

- **Build warning:** `"matchHostname", "matchPathname", "matchPort" and "matchProtocol" are imported from external module "@astrojs/internal-helpers/remote" but never used` — Internal Astro 6.0.8 issue, harmless, likely fixed in a future patch.
- **npm warning:** `Unknown project config "shamefully-hoist"` — From `.npmrc`, unrelated to the migration.

### Future Considerations

- **`astro-icon`** is effectively unmaintained (last release Dec 2024). If the overrides stop working, replace with a custom `Icon.astro` component using SVGs from `@iconify-json/lucide` directly, leveraging Astro 6's native SVG component support.
- **`astro-compress`** declares `astro: *` as a regular dependency (not peer dep), which is an anti-pattern. Monitor for conflicts on future Astro updates.
